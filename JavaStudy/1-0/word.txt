・IT
➡︎コンピュータとネットワークを利用した技術の総称でInformation Technologyの略。
・ICT
➡︎意味はITと変わらないがCommunicationの意味が追加されており個人の情報発信の意味が強調されている。
・IOT
➡︎身の回りのモノが全てインターネットが繋がっている仕組みのこと

・OS
➡︎アプリやデバイスを動作するための基本となるソフトウェアのこと。
・Windows
➡︎マイクロソフト社が開発したOSのこと。
・Mac OS
➡︎Apple社が開発したOSのこと。
・UNIX
➡︎現存するOSで最も古いOS。余計な機能が少なく安全性が高い。現在でも進化が進んでおり企業内や商品で使用されている。
・Linux
➡︎LinuxはオープンソースのOSであり改変や配布が可能。Linux同様に余計な機能が少なく安定的に動く。
・HTML
➡︎Hyper Text Markup Languageの略でウェブサイトを構築するために構築されたマークアップ言語。
・CSS
➡︎HTMLと組み合わせて使用する言語。HTMLがウェブページ内の各要素の意味や情報構造を定義するのに対して、CSSではそれらをどのように装飾するかを指定する。
・JavaScript
➡︎動的にコンテンツを更新したり、マルチメディアを管理したり、その他多くのことができるスクリプト言語。
・.NET（C#, VB）
➡︎マイクロソフト社が開発したプリケーション開発、実行環境。
・Java
➡︎プログラミング言語の一つで使用されている言語の中では世界中で１番実績のある言語。主なメリットとして処理速度が高速、プラットフォームに依存しない、オブジェクト指向があげられる。


・Java SE
➡︎Java言語でプログラミングを行う際に最低限必要な機能をまとめたもの。Androidアプリを作成する際にはJavaSEの導入は必須となる。
・Java EE (Jakarta EE)
➡︎JavaSEを元にしてサーバーサイドの機能を追加したもの。Webサイト、アプリケーション開発時に使用される。
・Perl
➡︎プログラミング言語としての歴史が長く利用者の多い言語。システムユーティリティやソフトウェアツールの作成、データベースへのアクセス、グラフィックやネットワーク処理、ウェブプログラミングなどに適している。
・PHP
➡︎HTMLと組み合わせて使用することができ、動的なコンテンツの作成に向いているプログラミング言語。E Cサイトやお問い合わせフォームを作成できる。
・Ruby
➡︎日本人が開発し初めて国際規格に認定されたオブジェクト指向言語。コードがシンプルで読みやすいのが特徴。
・Apache
➡︎Webサーバーソフトウェアで全世界のシェア５０％を占めているソフトウェア技術。
・nginx
➡︎WebサーバーソフトウェアでApacheに次いでシェア２位。最近注目されてきている。
・IIS
➡︎無料で使用できるWebサーバーソフト。Windowsで使用可能。
・SQL
➡︎データベースからデータの検索や追加、更新等を行う言語（データベース言語）。
一文ごとに結果を求めるのでプログラミング言語とは仕組みが違う。
・DB2
➡︎データーベース 管理システムのこと。データーベース言語として初めてSQLが採用された。
・MySQL
➡︎世界でもっとも利用されているデータベース管理システム。
・Maria DB
➡︎MySQLから派生したデーターベース管理システムで世界中で高速にシェアを伸ばしている。
・Oracle DB
➡︎オラクル社が開発販売を行っているデーターベース管理システム。企業のニーズを満たす拡張機能がある。
・PostgreSQL
➡︎通称ポスグレ。MySQLと人気を二分するデーターベース 管理システム。
・SQLite
➡︎簡易的なシステムのデーターベースでよく使用される。
・SQL Server
➡︎Windowsでよく使われるデーターベース 。
・CRUD
➡︎システムに必要な4つの主要機能である「Create（生成）」「Read（読み取り）」「Update（更新）」「Delete（削除）」の頭文字を並べた用語。
・トランザクション
➡︎処理のまとまりを表す用語
・コミット
➡︎トランザクション処理の結果を確定させる処理のこと。
・ロールパック
➡︎処理した結果を削除すること
・排他制御（排他ロック）
➡︎ダブルブッキングをしないようにするための処理
・楽観ロック
➡︎めったなことでは他者との同時更新は起きないであろう、という楽観的な前提の排他制御。
・悲観ロック
➡︎他者が同じデータに頻繁に変更を加えるであろう、という悲観的な前提の排他制御。
・実行計画
➡︎最も効果的に処理ができると判断されたクエリ実行結果を問い合わせから導き出すこと
・チューニング
➡︎処理時間や負荷の軽減のためにデータを調節すること
・パフォーマンス
➡︎性能のこと
・スロークエリ
➡︎データベースにおいて、時間のかかっているSQL(遅いSQL)のこと
・クエリログ
➡︎実行された全ての SQL クエリを出力してくれるログ。 SQL 実行エラーが発生した際に、どのような SQL が実行されたのかを正確に把握するために活用する。
・IDE （総合開発環境）
➡︎プログラムを作るために必要な物が全部詰まった開発環境。Visul Studioなど
・Eclipse
➡︎IDEの一つ。主にJavaの開発環境として使用されている。
・NetBeans
➡︎IDEの一つ。プロジェクト管理機能や入力支援機能を備えている。

・Spring ToolSuite
➡︎Eclipseをベースに、作られた開発ツール
・DBツール
➡︎データベース管理に使用できるツールの総称
・A5:SQL Mk-2
➡︎生産性を向上させるベく開発されたSQL開発ツール。
・pgAdmin
➡︎GUIによるPostgreSQLの管理ツール
・DBeaver
➡︎SQLクライアントツール
・MySQL Workbench
➡︎SQL 開発、サーバー設定、ユーザー管理、バックアップなどの包括的な管理ツールを提供するデータベース
・Sequel Pro
➡︎データベースをExcelのような表で確認・編集をすることができるMac専用のアプリ
・TeamSQL
➡︎データーベース をGUIで管理する為のソフトウェア
・PM
➡︎プロジェクトの運営、品質、納期等に責任を持ち、プロジェクトを円滑に推進させる役割を果たすプロジェクト管理者のこと
・PL
➡︎システムの構築・導入プロジェクトにおいて、プロジェクトチームを管理・統括する管理者のこと
・PG(PE)
➡︎コンピュータシステムのプログラムを作る人
・SE
➡︎顧客の要求から仕様を決定し、大まかな設計をするまでの情報システム開発における上流工程を行う人
・デザイナー
➡︎WebデザイナーはWebサイトのデザインを専門に手がける職種の人、またグラフィックデザイナーは印刷物や企業ロゴ、商品などで使用されるデザインを制作する
・エンジニア
➡︎エンジニアリングに対して専門的なスキルを有する人
・フロントエンド・エンジニア
➡︎Webサイトなどユーザーの目に触れる部分を作るエンジニアのこと


・サーバーサイド・エンジニア
➡︎バックエンド と意味に大差はないが基幹系企業で主に使われることが多い
・バックエンド ・エンジニア
➡︎サーバーサイド（Webサーバー側）やデータベースのシステムなど、ユーザーの目に見えない部分を作るエンジニアのこと
・UI
➡︎PC、スマホ、タブレットでユーザーの閲覧できる部分の全て（デザイン、フォント）
・CUI
➡︎キーボードに入力されるコマンドによって、OS上の操作を行う、ユーザーインターフェースの事を言う。コンピュータユーザーに対しての情報が、文字で表示される。ユーザーインターフェースには、文字を用いるCUI、とグラフィックベースを用いるGUIがある。
・GUI
➡︎グラフィカルユーザインターフェース(Graphical User Interface)の略語。コンピュータへ出す命令や指示等を、ユーザが画面上で視覚的に捉えて行動を指定できるもので、それまで主流であった命令文を入力して実行する方式CUIに比べ、直感的に操作できるのが特長。
・ファイル
➡︎プログラムファイル（OS、アプリ等）とデータファイル（スマホ写真、動画）の２種類がある。
・拡張子
➡︎ファイルの種類を区別させるためにファイルの末尾に付けられる文字列のこと
・ディレクトリ
➡︎ディレクトリはフォルダのことでWindowsでは分かりやすくフォルダと呼ばれている
・ルート・ディレクトリ
➡︎一番上のディレクトリ（フォルダ）のこと
・カレント・ディレクトリ
➡︎ユーザーが現時点で操作を行っているディレクトリのこと
・絶対パス
➡︎URLでページを指定して、目的地（情報）がどこにあるのかを確実に伝える（HTML)
・相対パス
➡︎今いる場所（階層）を基準にして、目的地（情報）がどこにあるのかを伝える（HTML)
・LAN
➡︎コンピューターや通信機器とケーブルや無線電波を接続し、相互にデータ通信ができるようにしたネットワークのこと。
・WAN
➡︎NTTなどの電気通信事業者が提供するWANサービスを利用して、本社 ⇔ 支社間など地理的に離れた拠点のLANを相互に接続するネットワークのこと

・クライアント
➡︎サーバーのサービスを受ける側のコンピューターを指す言葉。
・サーバー
➡︎何らかのサービスを提供する側のコンピュータを指す言葉。
・HTTP
➡︎ホームページのファイルを受け渡しするときに使うお約束事
・FTP
➡︎インターネットの線を使ってファイルを送受信する仕組み
・HTTPS
➡︎インターネットで暗号化通信をやるときに使うお約束事
・SSL
➡︎インターネット上でやり取りする情報を暗号化して送受信するための仕組み
・SSH
➡︎ネットワークを経由して他のコンピュータ（主にサーバ）に接続し、遠隔操作するための仕組みです。
・ポート、ポート番号
➡︎ネットワークとPCとの識別用番号
・HTTP、FTP、SSHの各ポート番号
➡︎HTTPは80、FTP20,21はSSHは22がポート番号
・ファイアウォール
➡︎不正アクセスから守るための防火壁
・TCP
➡︎安定性のある通信プロトコル
・IPアドレス
➡︎インターネットをするときにコンピュータに割り当てられる住所のこと
150.95.237.28のような数字で表記される
・グローバルIPアドレス
➡︎全世界で通用するIPアドレス
・プライベートIPアドレス
➡︎仲間内でしか通用しないIPアドレス
・ドメイン
➡︎IPアドレスを簡単で覚えやすい文字に置き換えたもの〇〇.com等
・DNS
➡︎IPアドレスとドメイン名の紐付けを行うシステム



・VPN
➡︎インターネット上に仮想の専用線を設け、安全なルートを確保した上で重要な情報をやり取りすることにより、盗み見や改ざんなどの脅威から大切な情報を守ることができます。無料WIFIなどに使われる

・ブリッジ
➡︎MACアドレス（ネットワーク機器に割り当てられた住所）を見てデータの行き先を振り分けてくれる機器
・プロシキサーバー
➡︎インターネット接続をできない（させない）内部ネットワークのPCとしてインターネット接続を行うサーバのこと
・SMTP
➡︎メール送信に関する通信プロトコル
・POP
➡︎メールサーバーに届いたメールを自分のパソコンにコピーしているときに使う通信プロトコル
・IMAP
➡︎メールをメールサーバ上で見るやり方
・セッション
➡︎通信の開始から終了までのこと
・クッキー
➡︎WebサーバがWebブラウザに渡すメモ
・キャッシュ
➡︎訪問したWEBページの情報を一時的に保存する仕組み。よく訪問するページは表示スピードが上がるというメリットがある
・インデックス
➡︎欲しい情報を探しやすくする為の仕組み
・フロー
➡︎「流れ」を意味する。データーフローならデーターの流れのこと
・タスク
➡︎仕事のこと
・Git
➡︎プログラミングやソースコードなどの変更履歴を記録・追跡するためのシステム
・SVN
➡︎ファイルの変更履歴を管理してくれるシステムのひとつ
・デグレ
➡︎「デグレード」の省略表現。デグレードとは設定変更によって今まで出来ていたことができなくなること。
・マージツール
➡︎ファイルに対して行われた変更を統合する管理システムのこと
